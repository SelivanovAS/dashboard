# Мониторинг судебных дел ПАО Сбербанк

Автоматизированная система мониторинга апелляционных дел с участием ПАО Сбербанк в суде ХМАО-Югры. Включает веб-дашборд для анализа дел, Python-скрипт для сбора данных с сайта суда, AI-дайджесты судебных актов и уведомления в Telegram.

**Дашборд:** [selivanovas.github.io/dashboard](https://selivanovas.github.io/dashboard/sberbank_dashboard.html)

## Возможности

- **Автоматический сбор данных** — ежедневный парсинг сайта суда `oblsud--hmao.sudrf.ru`
- **Интерактивный дашборд** — таблица дел с фильтрами, поиском и сортировкой
- **AI-дайджесты** — автоматическое резюмирование судебных актов через Claude API
- **Telegram-уведомления** — отправка дайджестов в чат
- **Аналитика** — статистика по ролям, исходам, категориям дел
- **Расписание заседаний** — блок ближайших заседаний с датами и временем
- **Отслеживание новых дел** — уведомление о появлении новых дел с момента последнего визита
- **Адаптивный дизайн** — десктоп (таблица) и мобильная версия (карточки)

## Стек технологий

### Фронтенд
- HTML5 / CSS3 / Vanilla JavaScript (single-page, без фреймворков)
- Шрифты: Literata, Source Sans 3, JetBrains Mono
- localStorage для хранения настроек и отслеживания новых дел

### Бэкенд
- Python 3.12
- `requests` — HTTP-запросы к сайту суда
- `html.parser` — парсинг HTML-таблиц
- Anthropic Claude API — генерация дайджестов
- Telegram Bot API — отправка уведомлений

### Инфраструктура
- Cloudflare Workers — точный запуск автоматизации по cron-расписанию
- GitHub Actions — выполнение workflow (парсинг, дайджест, коммит)
- GitHub Pages — хостинг дашборда
- CSV — хранение данных о делах

## Дашборд

Одностраничное веб-приложение, открываемое напрямую в браузере. Загружает CSV-данные из Google Sheets или напрямую из репозитория.

### Основные разделы

| Раздел | Описание |
|--------|----------|
| **Статистика** | 4 карточки KPI: всего дел, в производстве, процент выигрышей, опубликованных актов |
| **Чипы** | Разбивка по ролям (истец/ответчик/третье лицо) и результатам (оставлено/отменено/изменено) |
| **Ближайшие заседания** | До 10 ближайших заседаний с датой, временем, номером дела и сторонами |
| **Поиск и фильтры** | Полнотекстовый поиск, фильтры по статусу, роли банка и категории |
| **Таблица дел** | Сортируемая таблица с раскрывающимися строками и цветовыми бейджами |
| **Мобильные карточки** | Адаптивный вид для экранов < 768px |

### Фильтрация

- **Статус:** В производстве, Назначено, Отложено, Приостановлено, Решено, Архив (30+ дней)
- **Роль банка:** Истец, Ответчик, Третье лицо
- **Категория:** Динамически формируется из данных
- **Поиск:** По номеру дела, сторонам, категории, суду, событиям, заметкам

### Настройка источника данных

При первом открытии дашборд предлагает:
1. Подключить Google Sheets с CSV-данными (URL сохраняется в localStorage)
2. Загрузить демо-данные для ознакомления

## Бэкенд-автоматизация

Скрипт `scripts/update_cases.py` выполняет полный цикл обновления:

```
Сайт суда → Парсинг → Обновление CSV → AI-дайджест → Telegram → Git commit
```

### Этапы работы

1. **Загрузка текущего CSV** — чтение существующих данных из `data/sberbank_cases.csv`
2. **Парсинг поисковой выдачи** — запрос к сайту суда с поиском по "Сбербанк", разбор HTML-таблиц
3. **Обновление карточек дел** — загрузка детальной информации по каждому активному делу:
   - Последнее событие, статус, даты
   - Проверка публикации судебных актов
   - Извлечение мотивировочной части актов
   - Определение апеллянта
4. **Генерация дайджеста** — Claude API (Haiku) создает структурированное резюме новых актов
5. **Отправка в Telegram** — дайджест отправляется в указанный чат (с разбиением на сообщения до 4096 символов)
6. **Сохранение CSV** — обновленные данные коммитятся в репозиторий

### Особенности

- Фильтрация дел Сбербанк Страхование (дочерняя компания) — исключаются из мониторинга
- Вежливый парсинг — задержка 2–3 секунды между запросами
- Архивные дела (решённые 30+ дней назад) не обновляются
- Дедупликация дайджестов через файл `.digested_acts`

## Схема данных

Активные дела хранятся в `data/sberbank_cases.csv`, архив (дела решённые более 30 дней назад) — в `data/sberbank_cases_archive.csv`. У обоих файлов одинаковая схема — 20 колонок:

| Колонка | Тип | Описание |
|---------|-----|----------|
| Номер дела | Строка | Номер дела, напр. `33-2957/2026` |
| Дата поступления | Дата | Дата поступления в суд (ДД.ММ.ГГГГ) |
| Истец | Строка | ФИО / наименование истца |
| Ответчик | Строка | ФИО / наименование ответчика |
| Категория | Строка | Иерархическая категория дела (через `→`) |
| Суд 1 инстанции | Строка | Суд первой инстанции |
| Роль банка | Перечисление | `Истец` / `Ответчик` / `Третье лицо` |
| Статус | Перечисление | `В производстве` / `Решено` |
| Последнее событие | Строка | Текст последнего процессуального события |
| Дата события | Дата | Дата последнего события |
| Время заседания | Время | Время заседания (ЧЧ:ММ) |
| Акт опубликован | Перечисление | `Да` / `Нет` |
| Результат | Строка | Текст результата рассмотрения |
| Ссылка | Строка | `case_id\|case_uid` для ссылки на сайт суда |
| Заметки | Строка | Дополнительные заметки (ручной ввод через Google Sheets) |
| Апеллянт | Строка | Кто подал апелляцию (`Банк` / `Иное лицо`) |
| Дата публикации акта | Дата | Дата публикации судебного акта |
| Дата заседания | Дата | Дата ближайшего заседания |
| Судья 1 инстанции | Строка | ФИО судьи первой инстанции |
| Судья-докладчик | Строка | ФИО судьи-докладчика апелляционной инстанции |

### Архивирование

Скрипт `update_cases.py` автоматически переносит дела со статусом `Решено` и датой события более 30 дней назад в `data/sberbank_cases_archive.csv`. Это удерживает основной CSV компактным. Дашборд прозрачно подгружает оба файла и склеивает их (с дедупликацией по номеру дела) — пользователь видит полную историю.

## Запуск

### Дашборд (фронтенд)

Открыть `sberbank_dashboard.html` в браузере. Сборка не требуется.

Или использовать опубликованную версию: [selivanovas.github.io/dashboard](https://selivanovas.github.io/dashboard/sberbank_dashboard.html)

### Скрипт обновления (локально)

```bash
# Установить зависимости
pip install -r scripts/requirements.txt

# Задать переменные окружения
export ANTHROPIC_API_KEY="..."
export TELEGRAM_BOT_TOKEN="..."
export TELEGRAM_CHAT_ID="..."

# Полный цикл: парсинг + дайджест
python scripts/update_cases.py

# Только генерация дайджеста (без парсинга сайта суда)
python scripts/update_cases.py --digest-only
```

## Автоматизация

### Cloudflare Worker (триггер)

GitHub Actions cron ненадёжен на бесплатных планах (задержки до 3-4 часов). Поэтому запуск workflow выполняется через Cloudflare Worker, который вызывает GitHub API `workflow_dispatch` точно по расписанию.

- **Worker:** `court-monitor-trigger` (`cloudflare-worker/`)
- **Расписание:** Ежедневно в 3:50 UTC (6:50 МСК)
- **Секрет:** `GITHUB_PAT` — токен для вызова GitHub API

Изменить расписание можно:
1. В файле `cloudflare-worker/wrangler.toml` → `crons = [...]` → `npx wrangler deploy`
2. В Cloudflare Dashboard → Workers → court-monitor-trigger → Settings → Triggers

### GitHub Actions

#### Update Court Cases (`update_cases.yml`)

- **Триггер:** `workflow_dispatch` (через Cloudflare Worker или вручную)
- **Действия:** Парсинг сайта суда → обновление CSV → дайджест → Telegram → коммит в репозиторий

#### Digest Only (`digest_only.yml`)

- **Триггер:** Только ручной запуск (`workflow_dispatch`)
- **Действия:** Генерация дайджеста для ещё не обработанных актов → Telegram

## Переменные окружения

| Переменная | Описание | Где используется |
|------------|----------|------------------|
| `ANTHROPIC_API_KEY` | API-ключ Anthropic (Claude) | Генерация дайджестов |
| `TELEGRAM_BOT_TOKEN` | Токен Telegram-бота | Отправка уведомлений |
| `TELEGRAM_CHAT_ID` | ID чата/канала Telegram | Отправка уведомлений |
| `CSV_PATH` | Путь к активному CSV (по умолчанию `data/sberbank_cases.csv`) | Скрипт обновления |
| `CSV_ARCHIVE_PATH` | Путь к архивному CSV (по умолчанию `data/sberbank_cases_archive.csv`) | Скрипт обновления |
| `DIGESTED_ACTS_PATH` | Путь к файлу обработанных актов | Скрипт обновления |

В GitHub Actions переменные задаются через **Settings → Secrets and variables → Actions**.
